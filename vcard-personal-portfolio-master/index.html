    Object-Oriented Programming

    Object-oriented programming utilizes classes and objects to create models that have the ability to pass, receive messages, or process information in the form of data. Principles of object-oriented programming (Inheritance, Encapsulation, Abstraction, Polymorphism) are the pillars of OOP. “Models the world in objects” Object: An entity that has a well-defined role in the application domain, and it has state, behavior, and identity characteristics. Object Class: A logical grouping of objects that have the same (or similar) attributes, relationships, and behaviors; also called class. Benefits: Makes code more modular and easier to maintain.

    Encapsulation: restricts access to public methods by making instance variables private and accessor methods public. The technique of hiding the internal implementation details of an object from its external view.

    Abstraction: allows developers to focus on the essential features of an object and ignore irrelevant details. Involves selecting specific data from a larger pool to create a simplified representation of an object. This creates an abstract class that can be used as a blueprint for other classes to inherit shared characteristics. “A class that has no direct instance, but whose descendants may have direct instances.”

    Inheritance: expresses “is-a” or “has-a” relationships between two objects to create parent and child classes. In Java, class inheritance is used with “extends” or “implements” for interface inheritance.

    Polymorphism: The same operation may apply to two or more classes in different ways. Polymorphism uses methods inherited from other classes to perform different tasks.
